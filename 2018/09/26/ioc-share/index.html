浅谈依赖注入

前言

依赖注入，简而言之，在我们的代码中，依赖就是两个模块间的一种关联（在 Java 即指两个类）－－往往是其中的一个模块使用另外一个模块去做些事情。

谈依赖注入之前，先来了解一个概念“耦合度”。

耦合度(Coupling)是对模块间关联程度的度量。耦合的强弱取决与模块间接口的复杂性、调用模块的方式以及通过界面传送数据的多少。

软件设计中通常用耦合度和内聚度作为衡量模块独立程度的标准。划分摸块的一个准则就是高内聚低耦合。“低耦合，高内聚”是目前软件架构设计的重要问题之一，本文不详细阐述“低耦合，高内聚”的细节内容，留一点拓展的空间。

低耦合就是在软件设计过程中尽量的降低耦合度，也就是“解耦”，那么解耦和要聊的依赖注入到底有何关系？
